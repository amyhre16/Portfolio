Nomz

Nomz is a data-persistant, nutritional shopping list. Nomz provides data unique to the user. When an item is added to the list, its nutritional information is displayed in a table. Using the user's location, a map is displayed showing the stores closest to the user.
Trello was used to manage project tasks.
Collaborated with a group of 3 other developers.

Technologies Used

  • HTML5
  • CSS3
  • jQuery
  • Bootstrap
  • Express
  • Google Firebase
  • Edamam API
  • Google Maps API
  • Request NPM Package

Hot Restaurant

This restaurant waiting list app was built in 3 hours during class. A database is used to store reservations. Once there are 5 reservations, the table is added to a waiting list.
The user can view the reservation/waiting list and clear both lists at any time.

Trivia Game

This trivia game is a Texas Longhorn Football quiz. The questions, answers, and images are stored as objects in an array. Using a random number generator, the order of the questions and answers are randomly generated to keep the user on its toes. The user has 15 seconds to answer each question. After every question, the user is informed of whether or not its choice is correct. After all questions have been answered or the user runs out of time, the quiz results are displayed.
Worked independently

Technologies Used

  • HTML5
  • CSS3
  • jQuery
  • Bootstrap
App Link || GitHub Repository

Friend Finder

Friend Finder is a friend matching app. After filling out a survey, an algorithm matches the user with the person that had similar answers. The user's information is then added to the friends list so they can also be matched with new users. Friend Finder is not completely data persistent which means the user's data will not be saved forever.
